# MOOC DSP 讲义样稿：持久化需求与交接记录

## Objective

将 `MOOC课件-合并版.pdf` 制作为 A4 纵版、传统教材风的《数字信号处理讲义》；当前目标为完成并验收 5–10 页代表性样稿。

## Non-negotiable requirements captured on 2026-07-18

- 不漏原课件的唯一内容；沿用原章节、知识点和讲授顺序。只合并无信息损失的动画递进页，只删完全重复内容。
- 所有公式必须是真正的数学排版。下标、上标、求和、根号等不得以普通代码或缺字符号代替；除法一律使用上下结构的竖式分数，禁止用 `/` 表示除法。
- 公式框保留为全书统一样式；同一数学字体、同一基准字号。复杂分式只可增加行高，不可放大公式字号或溢出框体。
- 页面提示框标签使用“复习提示”，不使用“学习提示”。
- 原课件图可作为图本体使用，但必须移除课件顶部标题条、学校名称/标识、水印式元素、底部色条及来源说明；讲义自行在图上方添加图题。图内的曲线、箭头、公式、坐标和标注必须保持准确、清晰、无水印。
- 每次用户提出的新建议，都要立即记录在此类版本化文件中；每次上下文压缩或恢复后，先读此记录、交接记录、Git 状态和最新提交。
- 每个已验证的阶段须提交并推送 GitHub；不混入其他任务文件。

## Current implementation state

- 8 页样稿已生成，包含正文、原课件图、DFS 例题、课内练习、真题训练与详解。
- 数学排版已从普通文本改为数学渲染资产，并通过自动测试；旧版 Unicode 下标缺字问题已修复。
- 仍需完成：公式统一字号/数学字体的版式校正；原课件图的外围课件模板元素清理；逐页 300 dpi 视觉验收；将已验证的项目资料迁入独立仓库目录。

## Recovery protocol

1. 读取本文件及工作区 `work/mooc-dsp-handout-sample/findings.md`、`progress.md`、`task_plan.md`。
2. 检查本仓库的 `git status --short` 与 `git log -1 --oneline`；只处理本文件和本任务目录，保留无关用户改动。
3. 检查 `outputs/mooc-dsp-handout-sample/数字信号处理讲义_样稿.pdf` 的实际存在与最后修改时间，并重新渲染受影响页进行视觉验证。
4. 不得凭对话摘要宣称完成；以磁盘文件、测试和渲染结果为准。
